MyBatis执行流程-源码分析

发表于 2018-07-28 | 分类于 MyBatis | 阅读次数:

查看源码时最好用debug一步步调试

测试类

第一步: 首先先看SqlSessionFactory factory = builder.build(in);,即创建SqlSessionFactory 对象.

SqlSessionFactoryBuilder类

SqlSessionFactoryBuilder类

SqlSessionFactoryBuilder类

创建了一个 DefaultSqlSessionFactory对象,传了一个Configuration对象的参数.

Configuration是一个全局的配置对象,封装了所有的配置信息.

在这里看一下传的参数config,要看parser.parser()的底层

XMLConfigBuilder类

首先先看是否有没有解析,第一次解析,设置为ture后,又调用parseConfiguration方法,解析配置.

XMLConfigBuilder类

这里就在读取我们全局配置文件(主配置文件)中的信息,执行一系列操作之后一层一层返回,最后返回一个Configuration对象,该对象拥有所有的配置.再返回,返回到了SQLSessionFactory对象.可以看出,表面上创建了一个SQLSessionFactory对象,同时还创建了一个全局的配置对象.

第二步:SqlSession session = factory.openSession();

接下来再看一下SqlSession对象session是怎么创建的?

SqlSessionFactory接口的实现类–>DefaultSqlSessionFactory类

DefaultSqlSessionFactory类

这里有个事务,其实也不能猜到每次都要开启一个新的事务,SqlSessionFactory每次创建都创建一个新的,每个SqlSeesionFactory都有一个新的事务.

最后,它创建了一个DefaultSqlSession对象,用到了全局配置对象,Executor对象作为参数.

Executor对象是一个执行器,MyBatis的调度中心,负责SQL生成和查询缓存维护.在创建SQLSession对象之前创建完成.

在这里再看一下创建Executor对象的代码:

Configration类

在这里,它用到了一个CachingExecutor ,把默认的Executor对象executor包装起来了,

用Executor的子类CachingExecutor包装了一个SIMPLE,这是一个带有二级缓存的执行器.先去二级缓存中去寻找是否有数据.

其后有一个interceptorChain(拦截器链,多个拦截器的合称).其pluginAll方法是让每个拦截器调用plugin方法,相当于AOP,在做增强,包装.将executor做加强.然后就一步步返回到测试类中.

这里就是加载全局配置文件,创建SqlSessionFactory,创建SqlSession的整个过程.

第三步: UserMapper mapper = session.getMapper(UserMapper.class);

再看mapper对象是如何创建的?

先得到了一个MapperProxy对象,再调用newInstance方法.

通过JDK的动态代理机制创建一个Mapper的代理对象.也就是说通过

得到的也就是一个代理对象.

其中MapperProxy类中的invoke方法是具体的怎么做增强.

第四步: 接下来,我们具体看查询该怎么做?

它底层会去调用MapperProxy类的invoke方法,即代理对象的invoke方法.

它首先判断当前代理的方法listMap是不是Object类中的方法,如果是,就不做任何增强,直接执行.然后判断是不是默认方法,也不是.然后执行下面代码

其中,MapperMethod主要判断当前的方法是什么类型的方法.

我们测试类是select方法,在switch中找到select,首先判断该方法返回时是否为空并且是否有结果集处理器.是否会返回多条数据.可以如果是查询单个,用的是sqlSession.selectOne方法.

它会调用selectList方法,如果list中只有一个数据,会返回第0个,否则,报异常或者返回null.

selectList底层,会调用query方法.

其中,MappedStatement:映射语句对象,封装了一个元素节点(insert|update|delete|select)的信息.

首先会从二级缓存中去找,二级缓存中没有.会进入BaseExecutor(底层的执行器)

BaseExecutor会先从一级缓存localCache中去查询,如果没有再查询数据库的.

查询数据库的时候又会调用doQuery方法,

doQuery方法中会去创建一个StatementHandler对象

StatementHandler:语句处理器,封装了JDBC的DML/DQL操作,参数设置.其实就是一个SQL语句处理器.在创建对象的时候执行pluginAll方法.在创建StatementHandler对象之前,先创建参数处理器和结果集处理器.
